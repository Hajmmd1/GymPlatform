## Task Status

| Status | Description |
|--------|-------------|
| `todo` | Not started, ready for implementation |
| `in-progress` | Currently being worked on |
| `review` | Completed, awaiting review |
| `blocked` | Blocked by dependency or issue |
| `completed` | Done and verified |
| `cancelled` | No longer needed |

---

## Task Priority

| Priority | SLA | Description |
|----------|-----|-------------|
| `critical` | Immediate (ASAP) | Production issues, security vulnerabilities |
| `high` | Within sprint | Important features, technical blockers |
| `medium` | Next 2 sprints | Standard features and improvements |
| `low` | When capacity | Nice-to-have, technical debt |
